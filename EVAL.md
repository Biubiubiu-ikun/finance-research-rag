# 评测报告（Evaluation）

本项目坚持**评测驱动开发**：每一层的改动都用**留出测试集**量化，并**诚实记录负结果**与评测边界。

---

## 1. 检索层（200 题 / 8 行业留出集，无训练泄漏）

测试集由 `make_finance_eval.py` 生成，**排除 bge 训练用过的 chunk_id**（无数据泄漏）。

| 纯向量召回 | 通用 bge-small | 微调 bge-small-zh-finance | 提升 |
|---|---|---|---|
| R@1 | 22.5% | 35.5% | **+13pt** |
| R@5 | 43.5% | 60.5% | **+17pt** |

**泛化证据（13 个微调"未训练过"的新行业标的）**：微调 bge R@5 **60.5%** vs SOTA 通用 bge-m3(568M) **52%**；doc R@5 **75.5%** vs 69% → 是**领域泛化，不是过拟合**。

**完整混合检索**（BM25 + 微调向量 + 加权 RRF + 微调 reranker）：chunk R@1 59.5% / chunk R@5 79.5% / **doc R@5 86.0%**。微调 reranker 比通用 reranker chunk R@1 **+8pt**——粗排 bi-encoder + 精排 cross-encoder **两段都做了领域微调**。BM25 单路 R@1 **54.5%**（证明金融场景 BM25 不可或缺，纯向量会丢专有名词/代码）。

**跨模型横向对比**（60 题留出，纯向量 R@1）：

| 模型 | 参数量 | R@1 |
|---|---|---|
| gte-base-zh | 102M | 13.3% |
| 通用 bge-base | 102M | 25.0% |
| 通用 bge-small | 24M | 26.7% |
| SOTA 通用 bge-m3 | 568M | 36.7% |
| **微调 bge-small（in-batch）** | **24M** | **46.7%** |
| **微调 bge-small（+难负）R@5** | **24M** | **66.7%** |

→ **24M 微调碾压 568M SOTA**，"领域微调 > 堆 23× 参数"。

**诚实负结果**：难负样本挖掘重训后 R@5 微升（63.3→66.7）但 R@1 略降 1.7pt → 判定为**伪负噪声**（自动挖的难负里混进了真正例）。

## 2. 生成层（双重防幻觉）

- **数字级溯源**：~690 个财务数字逐字回原表核对，**100% 可溯源**，揪出 **2 处**抽取错误。
- **句级 entailment**：8 标的聚合结论逐条回原文做蕴含核查，**0 处**无依据幻觉。
- **LLM-judge**（`eval_generation.py`，20 题）：Faithfulness **86%** / Correctness **91%**。
- **评测驱动改进**：judge 抓出「Agent 把均值当中位数、漏报机构数」→ 根因是让 LLM 做**本该确定性**的统计 → 把中位数/家数放进**工具返回值** → 修正后 Correctness 100%。

---

## 3. Agent 层（调度质量）

- **`eval_agent.py`（45 题，含 17 越界）**：工具命中 **100%** / 越界拒答 **88%**（实时行情、知名库外标的等边界场景为已识别改进点）。
- **reflection 自检**：充分性判定 P/R 100%。
  > 说明：reflection 是**测试期自我修正**（LLM-judge 判完整性→不足则补调工具重答），**不是 RL、不是 reward**——别把它讲成强化学习。

---

## 4. 蒸馏（师生四方对照，独立 gold）

对照**独立构造标签** val（不是用老师自己的判断当 gold，破"自证"）：

| 选手 | 准确率 |
|---|---|
| base Qwen2.5-1.5B（未微调） | 70.4% |
| **学生 LoRA（1.5B + R1 思维链蒸馏）** | **92.0%** |
| 老师 deepseek-chat | 95.5% |
| 老师 deepseek-reasoner(R1) | 96.5% |

**诚实结论**：① 学生 92% 逼近老师、远超 base，蒸馏有效，且方法（R1 思维链蒸馏 + QLoRA + 师生评测）是完整的可迁移能力；② 但该模型**偏保守**（结论含"持续/接近"等非逐字措辞时易误判"不支持"），且 92% 系**同源 val**、会高估真实泛化；③ 句级核查是**低频任务**，本地小模型延迟高于 API → **结论是不值得用它替代 API**，故在系统里保留为**可切换引擎**而非默认。这是一个**有数据支撑的工程取舍**，不是"上线的更优方案"。

---

## 评测的边界（主动暴露）

- 生成 / Agent 评测（20 / 45 题）的 LLM-judge 自身有偏差；最严谨应扩到**人工标注黄金集**。
- 蒸馏 val 的 gold 为**构造标签**（proxy），非人工标注；R1 未参与造 val，故最独立。
- 上述局限均如实记录——这是"评测驱动开发"，不是"评测装点"。

---

## 复现

```bash
python make_finance_eval.py            # 生成 200 题留出集（排除训练块）
python eval_vector_compare.py          # 检索：多模型横向对比（R@k / doc 级）
python eval_reranker.py                # reranker：同候选池对比通用 vs 微调
python eval_generation.py              # 生成：LLM-judge faithfulness / correctness
python eval_agent.py                   # Agent：工具命中 / 越界拒答
python lora_finetune/eval_distill.py --all   # 蒸馏：师生四方 vs 独立 gold
```
